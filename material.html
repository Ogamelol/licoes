Introdução

A lógica de programação é a habilidade de organizar informações de forma sequencial e transmitir essas informações para o computador para que ele execute ações especificas. Ela é a base para a criação de software e instrui os computadores para executar tarefas de variada complexidade. Por conta disso, a lógica de programação é o primeiro passo para o indivíduo aprender os básicos de programação.

Importância da Lógica de Programação

  • Resolução de Problemas: Facilita a análise e decomposição de problemas complexos.
  • Organização: Um código bem estruturado é mais limpo e fácil de manter.
  • Prevenção de Erros: Uma lógica bem elaborada previne problemas futuros.
  • Facilidade de Aprendizado: Melhora o entendimento de linguagens e códigos.

Conceitos Básicos - Variáveis

Variáveis armazenam valores usados pelo programa. Alguns dos tipos mais comuns são:

  1. Inteiros: Armazenam números inteiros. Ex: 1, 2, 3...
  2. Reais: Armazenam números reais. Ex: 0.1, 1.5...
  3. Caracteres: Armazenam letras e palavras. Ex: 'a', 'abc'
  4. Lógico: Armazena um valor á base de verdadeiro ou falso.

Conceitos Básicos - Operadores

Operadores realizam ações sobre variáveis. Os operadores lógicos são responsáveis pelas variáveis lógicas, enquanto isso, os aritméticos tomam conta dos cálculos matemáticos. Eles são:

Operadores Tipo
Aritméticos + - * / %
Lógicos E OU Não NOR XNOR

Estrutura de Controle

As Estruturas de controle são necessarias para você estabelecer um fluxo no seu algoritmo para que ele crie condições para que o programa execute certas ações. Os dois mais importantes são:

  • Se (if): executa uma ação se a condição for verdadeira.
  • Senão (else): executa uma ação alternativa.

Um auxiliador extremamente importante para a criação do algoritmo seria a tag "leia", que vai ler o valor que o usúario imputar e adicionar ele a uma variavel.

Algoritmos

Um algoritmo é uma sequência de comandos para realizar uma tarefa. Exemplo de algoritmo simples:

x, y, z, r: real escreval("Qual o primeiro valor?") leia(x) escreval("Qual o segundo valor?") leia(y) escreval("Qual é a operação? (1 = +, 2 = -, 3 = *)") leia(z) se (z = 1) então r <- x + y escreval("Resultado da adição: ", r) senao se (z = 2) então r <- x - y escreval("Resultado da subtração: ", r) senao se (z = 3) então r <- x * y escreval("Resultado da multiplicação: ", r) senao escreval("Operação inválida") fimse fimse fimse

Neste algoritmo, nós podemos observar algumas particularidades. Inicialmente, foram criadas 4 variaveis, com X e Y sendo os valores que serão utilizados nas operações, Z sendo quem define qual operação será realizada, e R sendo a resposta da operação.

Logo em seguida podemos ver o "leia", que vai ler o valor que o usúario transmitir e repassar para as variaveis. Em seguida, nós utilizamos "se" para especificar o que deve ser feito dependendo da escolha do usúario, sendo Z quem determina isso.

E para finalizar, terminamos com "senao" que vai especificar todos os outros casos não específicados por "se". Para finalizar uma operação utilizando "se", precisamos usar "fimse", que sinaliza para a máquina que o "se" terminou. É necesssário pelo menos um fimse para cada "se" no código.
